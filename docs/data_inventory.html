Overview

This document provides a comprehensive inventory of all data files in the Muyumoqo Archaeological Project, including their structure, categories, and sample records. This reference ensures consistent data understanding across all analyses and collaborations.

1. Obsidian Geochemical Analysis

All data files are in data subfolder

File Information

  • Filename: obsidian_new_info.xlsx
  • Type: Multi-sheet Excel workbook (2 sheets)
  • Content: XRF (X-Ray Fluorescence) geochemical analysis data and archaeological provenience for obsidian artifacts

Sheet Descriptions

1.1 cal_obsidian

  • Purpose: Calibrated XRF geochemical data for obsidian artifacts and geological sources
  • Rows: 2,597 samples
  • Columns: 31
  • Key Variables:
    • File: XRF analysis file identifier (e.g., “01413-Obsidian”); links to xrf_no in location_obsidian
    • Name: Sample classification
      • “Muyumoqo” = Archaeological artifacts from the site (unknown provenance)
      • Other names (e.g., “Alca-5”) = Known geological obsidian sources
    • Elemental composition (ppm) with error values:
      • Mn, Mn Err: Manganese
      • Fe, Fe Err: Iron
      • Zn, Zn Err: Zinc
      • Ga, Ga Err: Gallium
      • Rb, Rb Err: Rubidium
      • Sr, Sr Err: Strontium
      • Y, Y Err: Yttrium
      • Zr, Zr Err: Zirconium
      • Nb, Nb Err: Niobium
      • Th, Th Err: Thorium
  • Use: Statistical matching of Muyumoqo artifacts to known geological sources based on elemental fingerprints

1.2 location_obsidian

  • Purpose: Archaeological context and provenience data for obsidian samples
  • Rows: 2,146 samples (final curated dataset)
  • Columns: 10
  • Key Variables:
    • xrf_no: XRF analysis file number (links to File in cal_obsidian)
    • sample_no: Sample ID (MOQ-001, MOQ-002, etc.)
    • unidad: Excavation unit number
    • cuad: Square/grid quadrant
    • rasgo: Associated archaeological feature — treat as nominal/categorical (each value represents a specific architectural feature; not sequential)
    • nivel: Depth values — treat as nominal/categorical for now (recorded in centimeters but not uniform/standardized; use as grouping variable rather than continuous depth)
    • excavador: Excavator initials
    • fecha: Date of excavation (format: DD-MM-YYYY)
    • weight (g): Sample weight in grams
    • notes: Contextual notes about recovery
  • Use: Link geochemical data to spatial and stratigraphic context for distributional analyses

Data Integration

To analyze obsidian sourcing with context: - Join key: File (cal_obsidian) = xrf_no (location_obsidian) - This links elemental composition to spatial and stratigraphic context - Workflow: 1. Filter cal_obsidian where Name = "Muyumoqo" to get artifacts 2. Filter cal_obsidian where Name != "Muyumoqo" to get geological sources 3. Join artifacts with location_obsidian for spatial context 4. Use statistical methods (e.g., discriminant analysis, PCA) to match artifacts to sources

2. Lithic Analysis

File Information

  • Filename: lithic_analysis.csv
  • Size: 99 KB
  • Type: CSV (comma-separated values)
  • Content: Comprehensive lithic (stone tool) analysis with morphological and technological attributes

Structure

Rows: ~300+ stone tool/debitage records

Columns (25 total):

Category Variables
Excavation Context Unidad, Cuad, Nivel, Rasgo, Excavador, Fecha, Context
Fragment Information N frag. (number of fragments), Tamaño frag. (fragment size)

Attributes % Cortex (cortex percentage), Portion (tool portion)

Key Recording Categories

  • Material types: Documented rock types used
  • Technological types: Classification of reduction sequences
  • Platform types: Striking platform morphology
  • Termination types: How flakes break (hinge, feather, step)
  • Cortex coverage: Original surface percentage on artifacts
  • Tool portions: Complete vs. broken tools
  • Modification status: Fresh vs. modified/used artifacts

2.2 Column definitions for lithic_analysis.csv

Detailed field-level definitions:

  • Unidad (Unit): Excavation unit where the fragment was recovered.
  • Cuad (Quadrant): Quadrant within the unit where the fragment was recovered.
  • Nivel (Level): Depth in centimeters where the fragment was recovered.
  • Rasgo (Feature): Associated archaeological feature; may be blank if the context is uncertain or no feature is associated.
  • Excavador (Excavator): Initials of the person who excavated and found the fragment.
  • Fecha (Date): Date of recovery/record.
  • Context: Ignore this field (not used for analyses).
  • N frag. (Number of fragments): Typically 1; values >1 indicate multiple very small fragments studied together.
  • Tamaño frag. (Fragment size): Integer representing square size (e.g., 1 = fits within 1×1 cm, 2 = fits within 2×2 cm).
  • Length, Width, Thickness: Measurements in centimeters; recorded only for special/interesting specimens (often blank).
  • Material: Material type code — here predominantly obsidian (code 16 in the codebook).
  • Technological Type: See the codebook (section 2.1) for numeric-to-description mappings.
  • Debris: See codebook; blank indicates the specimen is not classified as debris.
  • Platform Type: See codebook; blank indicates no visible platform.
  • Termination: See codebook; blank indicates no assessable termination.
  • % Cortex: Estimated cortex coverage as a percentage (0–100).
  • Portion: Flake/blade portion code (see codebook); may be blank if not assessable.
  • Modification: Modification code (see codebook); many entries are blank when not assessable.
  • Weight: Weight in grams.
  • Notes: Free-text notes field for contextual or analyst comments.
  • Foto: Marked with x when a photograph was requested or taken.

Notes: - Numeric codes referenced above are defined in the “Lithic code equivalences” section (2.1). - Missing or blank values indicate unrecorded or unassessable fields; perform cleaning/normalization prior to analysis.

3. Excavation Features & Contexts

File Information

  • Filename: muyumoqo_fichas_guide.csv
  • Type: CSV
  • Content: Feature-level (ficha) excavation documentation from Unit 5 (UE5)

Structure

Rows: ~80+ feature/context records

Columns (30 total):

Category Variables
Location Unidad, Cuad, Rasgo (feature), Nivel, Capa (layer)
Sediment Properties Codigo de Munsell, Munsell Color, Textura (soil texture)
Feature Description Rasgos, hallazgos y estructuras (features, findings & structures)
Artifact Presence (X/blank) Cerámica, Lítica, Obsidiana, Batán/mortero/mano, Hueso animal, Hueso humano, Restos botánicos, Malacológicos, Metal, Arcilla Quemada, Yeso, Adobe, Otros
Sampling Carbon Muestras, Suelo Muestras, Cantidad total de bolsas
Field Metadata Herramientas (tools), Otro (other), Excavador, Fecha, Notes

Recording Method

  • Presence indicators: Marked with “X” if artifact category present in feature
  • Artifact types: 13 distinct material categories recorded
  • Detailed descriptions: Spanish-language feature descriptions capturing stratigraphy, finds, and interpretations
  • Special finds: Noted in Rasgos, hallazgos y estructuras column (e.g., diorita verde from Saqsaywaman, worked bone, textiles tools)
  • Samples: Distinct carbon and soil samples collected by bag number

Notable Features Documented

  • Wall falls and collapses
  • House interior/exterior deposits
  • Possible guinea pig enclosures (cuyerías)
  • Hearth/fire features (fogón) with ash and charcoal
  • Evidence of textile production (tools, dyes)
  • Foreign materials (Saqsaywaman diorite)

Data Integration Guide

Cross-referencing Between Datasets

Obsidian ↔︎ Lithic Analysis: - Link via: Unidad, Cuad, Nivel, Rasgo, Excavador, Fecha - The obsidian sample_no (MOQ-###) does correspond to collection/bag numbers

Obsidian ↔︎ Excavation Features: - Link via: Unidad, Cuad, Rasgo (feature), Nivel - Features in fichas sheet provide contextual interpretation for artifact assemblages

Lithic Analysis ↔︎ Excavation Features: - Link via: Unidad, Cuad, Nivel, Rasgo - Features document depositional context
